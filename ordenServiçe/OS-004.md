### 📜 ORDEM DE SERVIÇO: OS-004 (Operação Vitrine de Cristal)

**De:** Arquiteto-Chefe / Comando Geral
**Para:** Esquadrão de Código
**Alvo:** `c:\Users\Ti\Desktop\neuralsafety_rag_showcase`
**Objetivo:** Estruturar o ecossistema RAG Standalone, forjar as dependências exclusivas de IA e padronizar as variáveis de ambiente para demonstrações comerciais.

#### 🏗️ Fase 1: A Arquitetura de Pastas (O Padrão Ouro)

O repositório deve refletir a seguinte topografia exata. O Esquadrão deve garantir que nada fuja dessa estrutura:

```text
/neuralsafety_rag_showcase
  ├── /core/                   <-- O Cérebro
  │    ├── rag_service.py      <-- Lógica vetorial e chamadas LLM
  │    └── memory_manager.py   <-- Gestão de histórico de chat
  │
  ├── /data/
  │    └── /vector_db/         <-- Onde o ChromaDB salva o .sqlite3 (IGNORADO NO GIT)
  │
  ├── /static/                 <-- A Interface (HTML/CSS/JS do chat)
  │
  ├── /scratch/                <-- Scripts de teste e laboratório
  │
  ├── agent_rag.py             <-- O Ponto de Ignição (Servidor/Interface Principal)
  ├── requirements.txt         <-- [NOVO] O arsenal de bibliotecas
  ├── .env.example             <-- [NOVO] O mapa de chaves
  └── README.md                <-- [NOVO] Instruções de como rodar a demo
```

_(Nota tática: Garantam que o arquivo `.gitignore` inclua a pasta `data/vector_db/` e `.env` para não vazarmos dados de clientes ou chaves da OpenAI no GitHub)._

#### 📦 Fase 2: O Arsenal de IA (`requirements.txt`)

#### 🔐 Fase 3: As Chaves de Ignição (`.env.example`)

O sistema agora consome uma API externa (o nosso Batalhão). Crie um arquivo chamado `.env.example` para que qualquer engenheiro saiba o que preencher antes da demo:

```env
# 🧠 Cérebro do Agente (OpenAI)
OPENAI_API_KEY=sk-sua-chave-aqui

# ⚡ Conexão com o Batalhão (WebFetch API)
# Em dev: http://localhost:8000/v1/agent/search_and_fetch
# Em prod: https://api.neuralsafety.com/v1/agent/search_and_fetch
NEURALSAFETY_API_URL=http://localhost:8000/v1/agent/search_and_fetch
NEURALSAFETY_API_KEY=dev-key-001
```

#### ⚙️ Fase 4: Protocolo de Inicialização (Comandos de Terminal)

Para garantir que a base está limpa e operante, o Esquadrão deve abrir o terminal (`PowerShell` ou `CMD`) dentro da pasta `neuralsafety_rag_showcase` e executar a sequência de blindagem:

**1. Criar o Ambiente Virtual Isolado (VENV):**
Isso garante que as bibliotecas do RAG não entrem em conflito com as bibliotecas globais do Windows.

```powershell
python -m venv .venv
```

**2. Ativar o Ambiente Virtual:**

```powershell
# No Windows PowerShell:
.\.venv\Scripts\Activate.ps1
# (Se der erro de permissão, rodar: Set-ExecutionPolicy Unrestricted -Scope CurrentUser)
```

**3. Instalar o Arsenal:**

```powershell
python -m pip install --upgrade pip
pip install -r requirements.txt
```

**4. Criar o arquivo `.env` real (cópia do exemplo):**

```powershell
copy .env.example .env
```

_(Após isso, o operador deve abrir o `.env` e colocar sua chave real da OpenAI)._
